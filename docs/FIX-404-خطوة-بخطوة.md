# حل 404 عشان الشكل الجديد يشتغل طبيعي

## الوضع باختصار

| الجزء | الحالة |
|--------|--------|
| **التطبيق (الشكل الجديد)** | جاهز — بيطلب `https://api.easytecheg.net/trpc/...` |
| **الـ Backend (router.php)** | جاهز — موجود في `/mnt/marichia/files/easytech-new-api/backend/` وبيقبل `/trpc/` و `/api/trpc/` |
| **الداتا (عملاء، منتجات، مستخدمين)** | كلها في قاعدة البيانات — مش اتمسحت |
| **المشكلة** | الويب سيرفر اللي بيخدم `api.easytecheg.net` **مش موجّه** طلبات `/trpc/` لـ `router.php`، فبيرجع 404 |

يعني: **محتاج نضبط السيرفر مرة واحدة** عشان أي طلب لـ `/trpc/...` يروح لـ `router.php`. بعد كده الدخول والداتا هتظهر طبيعي.

---

## الخطوة الوحيدة المطلوبة: إعداد الويب سيرفر

المسار على السيرفر: **`/mnt/marichia/files/easytech-new-api/backend/`**  
الملف اللي لازم يشتغل: **`router.php`**

### لو عندك Nginx (الأكثر شيوعاً)

ادخل على السيرفر وافتح إعداد الـ server block بتاع `api.easytecheg.net` (غالباً في `/etc/nginx/sites-available/` أو `/etc/nginx/conf.d/`).

تأكد إن عندك شيء من دول:

**اختيار 1 — الـ root = مجلد الـ backend وتوجيه كل الطلبات لـ router.php:**

```nginx
server {
    listen 443 ssl;
    server_name api.easytecheg.net;

    root /mnt/marichia/files/easytech-new-api/backend;
    index router.php;

    location / {
        try_files $uri $uri/ /router.php?$query_string;
    }

    location ~ \.php$ {
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_pass unix:/run/php/php-fpm.sock;   # أو 127.0.0.1:9000
    }
}
```

**اختيار 2 — لو الـ root مكان تاني، ضيف location لـ /trpc/ فقط:**

```nginx
location ~ ^/(api/)?trpc/ {
    rewrite ^/(api/)?trpc/(.*)$ /router.php last;
}
# واتأكد إن الـ root بيوصل لمجلد الـ backend أو استخدم alias
```

بعد التعديل:
```bash
sudo nginx -t && sudo systemctl reload nginx
```

### لو عندك Apache

في الـ VirtualHost بتاع `api.easytecheg.net` أو في `.htaccess` داخل مجلد الـ backend:

```apache
RewriteEngine On
RewriteBase /
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^(api/)?trpc/.* router.php [L,QSA]
```

بعد التعديل:
```bash
sudo systemctl reload apache2
```

## اختبار من الطرفية (على السيرفر أو من جهازك)

بعد ما تضبط الإعداد، جرّب:

```bash
curl -s -o /dev/null -w "%{http_code}" "https://api.easytecheg.net/trpc/auth.adminLogin?batch=1"
```

- لو الناتج **200** يبقى التوجيه شغال (حتى لو الرد JSON فيه خطأ صلاحيات، المهم مش 404).
- لو لسه **404** يبقى الإعداد مش واصّل لـ `router.php` — راجع الـ root والـ location.

## بعد ما الـ 404 تختفي

1. افتح **https://api.easytecheg.net/app**
2. اعمل **Ctrl+F5** (تحديث قوي)
3. سجّل دخول — الداتا (عملاء، منتجات، مستخدمين) هتظهر من قاعدة البيانات كما هي

لو الويب سيرفر على TrueNAS أو داخل Docker/Proxy، نفس الفكرة: أي طلب لـ `api.easytecheg.net/trpc/...` لازم يوصل لـ `router.php` في مجلد الـ backend.
